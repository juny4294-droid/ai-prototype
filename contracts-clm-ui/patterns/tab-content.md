# Tab Content Baseline

- Source: ContractS CLM 契約書詳細画面の right panel タブ構成
- Purpose: tab title・section order・action placement・state transition の基準
- 参照元: 旧 `baselines/tab-content.pdf`（削除済み）

---

## タブ一覧と順序

1. 概要
2. 締結
3. 契約審査
4. 承認
5. コメント
6. 関連契約書
7. 編集ルーム
8. ログ

---

## 各タブ仕様

### 1. 概要タブ

**Header actions**: 編集

**Sections（順序固定）**:
- 契約書の概要（説明テキスト）
- 契約期間
  - 開始日・終了日
  - 自動更新（あり / なし）
  - 自動更新の更新拒絶期限
  - 更新に関する補足コメント
- 担当者
- フォロー中のメンバー
- 当事者（契約上の立場・会社名・住所・代表者名）
- 共有ファイル（ファイルアップロード）
- ログ（操作履歴）

---

### 2. 締結タブ

**States**:

#### State A: 未送信（初期）
- 「文書をDocuSignに送って締結をします」等の説明文
- **締結手段**選択（必須）:
  - DocuSign
  - ContractS SIGN
  - クラウドサイン
  - 自社からの締結依頼 / 紙
  - 自社からの締結依頼 / その他の電子契約サービス
  - こちらからの締結依頼 / 相手からの締結依頼
- Header actions: `締結依頼`

#### State B: 締結依頼フォーム（DocuSign選択時）
- 封筒に同封する文書を選択
- 言語（必須）
- ページサイズ・カラー・解像度 等 PDF 情報表示
- 押印設定（押印あり / 押印なし）
- 押印位置の設定（位置設定 / 設定しない）
- 署名者（甲・乙など各ステップ）
- ステップの追加
- Actions: `次へ` / `キャンセル`

#### State B': 締結依頼フォーム（ContractS SIGN選択時）
- 封筒に同封する文書を選択
- 言語（必須）
- 押印依頼方法 URL
- 押印位置: 設定しない
- 署名者
- Actions: `次へ` / `キャンセル`

#### State C: 送信確認
- 送信内容の確認画面
- Actions: `この内容で送信` / `最初からやり直す`

#### State D: 締結フロー確認（送信後）
- 締結作業フロー表示（例: 1. 押印ステップ）
- ステップごとの状態（完了 / 未完了）
- 締結済み契約書(PDF) アップロード欄
- 「まだ送信は完了していません。内容を確認して、送信してください。」等の案内文
- 利用期限（ContractS SIGN）
- Actions: `最初からやり直す`

**締結フロー URL 表示**:
- `【締結前】締結依頼 URL` + コピーボタン
- `【締結後】締結済み契約書(PDF)アップロード`

---

### 3. 契約審査タブ

**States**:

#### State A: 依頼なし
- 「依頼済みの契約審査はありません」表示
- Actions: なし（または `再度、契約審査をお願いします` リンク）

#### State B: 依頼フォーム
- 審査の種類（例: 法務相談）
- 件名（必須）
- 期限
- 内容
- 担当者（必須）
- 同時に審査をする契約書（関連契約書から選択）
- 作成中の関連契約書（同時依頼の選択）
- Actions: `依頼` / `キャンセル`

#### State C: 審査中
- 依頼日・依頼者・ステータス（未完了 / 進行中）
- 審査項目セット名
- 編集制限設定（担当者または依頼者のみ編集を許可する）
- 期限・説明
- Actions: `削除` / `タスクを完了`

---

### 4. 承認タブ

**States**:

#### State A: 承認フロー未選択
- 承認フロー一覧（例: 「事業本部_押印承認フロー」）
- フローの説明文
- Actions: `承認を申請` / `次へ` / `キャンセル`

#### State B: 承認申請フォーム
- 承認フロー名（必須）
- 件名（必須）
- コメント
- 締結フロー
- 同時に承認依頼をする契約書（チェックボックス）
- 同時に審査をする契約書
- Actions: `申請` / `キャンセル` / `次へ`

#### State C: 承認進行中
- 現在の承認ステップ表示（例: 1. 事業部承認者 → 2. 法務部承認者）
- 各ステップの承認者名・所属・ステータス（承認待ち / 完了）
- 承認が未完了のため選択できない場合の案内
- 同時に承認依頼が届いている関連契約書の一括承認チェックボックス
- Actions: `差戻し` / `承認` / `完了`

---

### 5. コメントタブ

**表示項目**:
- コメント一覧（投稿者・所属・日時・本文）
- @メンション対応

**Input**:
- テキストエリア: 「コメント（@メンションで相手に通知）」
- Actions: `投稿`

---

### 6. 関連契約書タブ

**Header actions**: `＋ 関連契約書の追加`

**List items** (各カード):
- 契約書名
- 元契約（プロジェクト名等）
- 識別番号（ab1000349 等）
- 契約日

---

### 7. 編集ルームタブ

**Header actions**: `＋ 編集ルームを新規作成`

**Sections**:
- 進行中のルーム
  - ルームメンバー一覧
  - `ルームを開く`
  - 申し送り事項
- 完了したルーム
  - `ルームを開く`

---

### 8. ログタブ

**表示形式**: タイムライン

**各エントリ**:
- 操作者名・日時
- アクション内容（例: 「業務委託契約 の締結依頼を承認しました。」）
- 元のファイル参照（ファイル修正時）

**ログに記録される操作例（順序は時系列降順）**:
- タスクを作成・完了
- 契約項目を変更
- 承認フローを発行・完了
- 締結依頼を発行・承認
- 自動更新
- 締結済みファイルを修正
- 更新契約を作成

---

## Anti-patterns（禁止事項）

- タブ順序を指示なしに入れ替えない
- 各タブの state を勝手に統廃合しない
- section title・inline action・footer action の配置を入れ替えない
- PDF baseline があるタブでは `empty / create / detail / progress` の state 数を勝手に減らさない
